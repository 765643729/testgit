Gameplay in 2D

虽然Unity是因为它的3D而著名，但它也可以用来创建2D游戏。可以在编辑器找到相似的功能来帮助单纯的2D的开发。

最值得关注的是在Scene界面下的工具栏的 2D 视图模式（2D view mode）按钮。 当激活 2D 模式时, 会放置一个正交视图（也就是perspective-free）;相机沿着z轴和Y轴向上增长。这可以使你可以更容易的具体化场景和放置2D对象。

要了解所有的2D组件，怎样选择2D还是3D模式，以及2D模式和3D模式的区别，请参考 2D or 3D Projects.

2D 图形（Graphics）

2D图形对象被称为精灵（Sprites）。精灵本质上只是标准纹理，但是在开发过程中有特别的可以提高效率和便利性的组合和管理精灵纹理的技术。Unity 提供一个内置的 精灵编辑器（Sprite Editor） 以供从一个大的图像中提取部分作为精灵图像。它使你能够在你的图形编辑器中把图片的一部分作为单一的纹理来编辑。比如，你可以把图片中一个角色的手、脚和身体拆分为单独的元素。

精灵是用 Sprite Renderer 组件渲染而非像3D对象那样使用 Mesh Renderer。你可以通过组件菜单(Component > Rendering > Sprite Renderer)将它添加到GameObject，或者，你可以使用已经附加上的Sprite Renderer直接创建一个GameObject(目录: GameObject > 2D Object > Sprite)。

另外，你可以使用 Sprite Creator 工具制作一个占位符（placeholder）2D图像。

2D 物理学（Physics）

Unity有单独的用于处理2D的物理引擎，仅用于2D的优化。这些组件对应标准的3D物理组件，例如刚体, 箱体碰撞器（Box Collider）和链条连接（Hinge Joint），但是命名上加上了“2D”。因此，精灵可以配备刚体2D（Rigidbody 2D），箱体碰撞器2D（Box Collider 2D）和链条连接2D（Hinge Joint 2D）。大多数2D物理组件就是3D的“扁平化”版本（例如，箱体碰撞器2D（Box Collider 2D）是一个长方形，而箱体碰撞器（Box Collider）是一个立方体）但也有一些例外。

关于2D物理组件的完整列表，请查阅2D or 3D Projects。更多关于2D物理概念和组件的信息请查阅Physics章节。
